## Ключевой контекст (пригодится дальше)

- **Источник истины — код, а не доки.** Этот проход сверял каждый claim против
  `sql/*`, `airflow/dags/*`, `configs/default_user.xml`, `docker-compose.yml`. Подход себя
  оправдал — так и нашлись расхождения. Повторять при любых правках доков.
- **Канон по слоям — уроки 2–3 курса**, а не ARCHITECTURE. В этой сессии ARCHITECTURE
  подтянут к их формулировкам (DQ-split как пересекающийся, dds.event как browser-driven
  LEFT JOIN). Уроки 2–3 сверены с кодом и точны; уроки 0,1,4,5,6 **не** проверял.
- **Рамка путей запуска (важно, владелец подчёркивал):** основной путь — **Airflow**
  (DAG-и `ddl_init`/`kafka_load`/`etl_pipeline`); `scripts/*` и обёртки `make` — **запасной
  fallback** в обход Airflow. Старый README до правки прямо помечал `make ddl/transform`
  как «(вне Airflow)», а `scripts/` — «legacy fallback». Эта рамка теперь зафиксирована в
  REPO_MAP; держать её во всех доках, не возвышать Makefile.
- **Параллельный Codex в ветке.** Codex пишет другие уроки в ту же ветку. Git держать
  строго аддитивно: `git add` только своих файлов, не amend/rebase/reset чужих коммитов.
  (Свой не-запушенный коммит amend-ить можно — в этой сессии так и поправил оплошность с
  индексом.)
- **Стиль:** чистый литературный русский (память `plain-literary-russian`); house style
  осознанный (тире «—», «ё», жирное начало абзаца) — линтером не править. `ai-text-lint`
  гонять только на настоящие маркеры, context `article`.
